> 总时长 12-15 分钟 | 格式：代码 + 桌面录屏 + 画外音

---

## 第一段：项目简介（0:00–1:00）

**画面**：应用窗口全屏展示，鼠标缓慢划过屏幕

**画外音**：
```
这是一个基于 Qt 6 + QML 的跨平台桌面天气应用——QWeather。
它集成了和风天气的全部 24 个 API 端点，从实时天气到分钟级降水，
同时利用 GPU 着色器实时渲染动态天空背景。
```

**字幕要点**：
- 技术栈：Qt 6.11 / QML / C++17 / GLSL
- 数据来源：和风天气 API + SQLite 缓存
- 核心亮点：V3 动态天空模拟系统

## 第三段：架构总览（4:00–5:30）

**画面**：全屏展示架构图 SVG，按 7 层逐层高亮出现（从顶到底）

**画外音**（分段推进，每层停顿 3-5 秒）：

```
[第一层·API 层]
最顶层是和风天气 API。项目封装了 24 个 HTTP 端点，覆盖地理、天气、
空气质量、天文、太阳辐射、台风、海洋等全部数据接口。

[第二层·WeatherAPI C++ 层]
第二层是 WeatherAPI，C++ 编写的统一网络层。所有的 API 请求经由
sendRequest 发出，在 onReplyFinished 中统一回收入口，解析响应类型，
分发到对应的 handler 函数，同时自动写入 SQLite 缓存。

[第三层·缓存与数据层]
第三层分为三个模块：左侧的 WeatherCache 用 SQLite 做键值存储，
自带 TTL 过期机制；中间是 4 个数据 Store——预报、空气质量、
太阳天文、城市详情，负责把原始 JSON 预处理成 QML 可以直接消费的格式；
右侧是 GlobalClock 和 AppSettings，提供 60 帧时间源和用户设置。

[第四层·QML 引擎]
第四层是 QQmlApplicationEngine。main.cpp 中通过 10 次
setContextProperty 把 C++ 对象注入到 QML 的全局命名空间，
从此 QML 可以直接调用 weatherApi.searchCity() 这样的 C++ 方法。

[第五层·天空系统]
第五层是 V3 天空模拟系统。BackgroundManager 是中央控制器，
连接天文模型、天气码配置和过渡编排器。AstronomyModel 计算太阳月亮位置，
WeatherProfile 把天气码映射成渲染参数，TransitionController 协调各图层的
淡入淡出时序。

[第六层·QML 页面]
第六层是 7 个 QML 页面。Main.qml 管理侧边栏导航和城市路由，
每个页面只负责布局和交互，数据全部来自 C++ Store 的属性绑定。

[第七层·着色器]
最底层是 WeatherBackground 的 6 层 ShaderEffect 栈。
从底到顶依次是天空渐变、大气散射、体积云、雨雪粒子、
雾和闪电。7 个 GLSL 着色器通过 qt_add_shaders 编译为 .qsb 格式，
在 GPU 上以 60 帧运行，最后经过 ACES 色调映射输出。
```

**过渡到下一步**：
```
整个架构的核心设计原则是：C++ 管数据，QML 管 UI，GLSL 管渲染。
三层各司其职，数据单向流动。
接下来我们看一下代码层面是如何实现这个架构的。
```

## 第四段：天空系统核心——C++ 控制器（5:30–8:30）

### 4.1 SkyState 数据契约（5:30–6:00）

**画面**：展示 `SkyState.h`，高亮 20 个字段分组

**画外音**：
```
SkyState 是一个 Q_GADGET 结构体，20 个字段分为天文/大气/天气/变体 4 组。
所有字段通过 Q_PROPERTY 直接暴露给 QML，ShaderEffect 的 uniform 绑定到这些属性上。
```

### 4.2 AstronomyModel 天文计算（6:00–6:30）

**画面**：展示 `AstronomyModel.cpp` 中 `calcSolarPosition()`

```
简化天文公式：
  太阳赤纬 = 23.45° × sin(2π × (dayOfYear - 81) / 365)
  时角 = (当前分钟 - 720) × 0.25°
  高度角 = asin(sin(lat)×sin(decl) + cos(lat)×cos(decl)×cos(ha))
```

### 4.3 BackgroundManager 大气颜色模型（6:30–7:30）

**画面**：展示 `buildAtmosphereChanges()` 的 7 段表格

**画外音**：
```
根据 dayProgress 在 7 个颜色段之间插值：
深夜（深蓝）→ 蓝调时刻 → 金色时刻 → 白天（天蓝）
→ 暖午后 → 橙红 → 紫色。

同时分段计算曝光度、星星可见度和黄昏因子。
```

### 4.4 TransitionController 图层编排（7:30–8:30）

**画面**：展示 `activateLayer()` 和 `deactivateLayer()` 的时序图

```
激活时序：0ms 触发 → 100ms 创建 Loader → 116ms tp=1 → 716ms 动画完成
去激活时序：tp=0 → 600ms Behavior 淡出 → 销毁 Loader
```

**画外音**：
```
TransitionController 负责协调 4 个条件图层的激活/去激活。
通过迟滞阈值防止振荡，延迟激活保证云→雨→雾的正确叠加顺序。
```

## 第五段：天空系统核心——QML 图层与着色器（8:30–10:30）

### 5.1 图层栈架构（8:30–9:00）

**画面**：展示 `WeatherBackground.qml` 的 z-order

```
z:-1  WeatherBackground
  z:0   SkyLayer (始终渲染)
  z:10  AtmosphereLayer (始终渲染)
  z:20  Loader → CloudLayer
  z:30  Loader → RainLayer / SnowLayer
  z:40  Loader → FogLayer
  z:50  Loader → LightningLayer
```

### 5.2 sky.frag 着色器（9:00–9:45）

**画面**：录屏中暂停，放 `sky.frag` 代码，标注关键段

```
1. 天空渐变：mix(zenith, horizon, pow(uv.y, 0.8))
2. 太阳光晕：exp(-dist² × 15) × 暖色
3. 月亮 SDF 月相裁剪
4. 星星：50 个粒子，正弦闪烁
5. ACES 色调映射
```

### 5.3 cloud.frag 体积云（9:45–10:30）

**画面**：代码 + 实际云层渲染效果并排展示

```
Inigo Quilez 算法：
1. 3 层 fBM 噪声 + 域扭曲
2. Emptiness/Sharpness 重映射
3. 4 步光线步进模拟体积感
```
