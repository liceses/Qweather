# 天气天空模拟系统 —— 设计文档 (V3)

> Qt 6.11 + QML + ShaderEffect · Layer 架构 · 生产级
> 替换旧版 indicationsForDynamic_background.md (V1~V2.2)
>
> **核心原则**：计算 & 数据管理 → C++ 端；纯渲染 → QML 端。
> 遵循项目现有模式：C++ Store + QML 纯渲染 + context property 注册。

## 一、架构总览

### 1.1 C++ / QML 边界

```
C++ 端 (数据 + 计算 + 状态)             QML 端 (纯渲染 + 简单 UI)
─────────────────────────────────      ─────────────────────────
GlobalClock (QTimer)              →    GlobalClock.elapsed
WeatherProfile (code→物理状态)     →    (由 Manager 消费, QML 不接触)
AstronomyModel (太阳/月亮位置)     →    Manager.skyState.solarAltitude ...
SkyState (统一渲染状态 struct)     →    各 Layer 绑定 manager.skyState.*
BackgroundManager (状态维护+模式)   →    各 Layer 绑定 manager.*
TransitionController (状态机)      →    Layer Loader.active 绑定 ctrl.cloudActive ...
                                       WeatherBackground (Layer 栈容器)
                                       SkyLayer / AtmosphereLayer / CloudLayer ...
                                       WeatherBackgroundDebugPanel
                                       [现有 UI: RowLayout + StackLayout]
```

### 1.2 Layer 栈 (QML 端)

```
ApplicationWindow
├── WeatherBackground                  ← Layer 栈容器 (QML Item)
│   ├── z=0   SkyLayer                ← 太阳 / 月亮 / 星星
│   ├── z=10  AtmosphereLayer         ← twilight / 散射 / 曝光 / 地平线色调
│   ├── z=20  CloudLayer              ← fBM 云噪声 (条件, 由 C++ TransitionController 控制)
│   ├── z=30  WeatherLayer            ← Rain | Snow 粒子 (条件)
│   ├── z=40  FogLayer               ← 雾 / 霾 / 沙尘 (条件)
│   └── z=50  LightningLayer          ← 全屏闪电闪烁 (条件)
├── [现有 UI: RowLayout + StackLayout]
└── WeatherBackgroundDebugPanel       ← 浮动面板, z=100
```

### 1.3 数据流

```
Main.qml focusId 变化
  → weatherApi.weatherNow(focusId)        // QML → C++ API (已有)
  → weatherApi.astronomySun(focusId, ..)  // QML → C++ API (已有)
  → weatherApi.astronomyMoon(focusId, ..) // QML → C++ API (已有)
       ↓
Main.qml onWeatherNowReady(now):
    backgroundManager.updateWeather(now.icon, isDay) ← 仅传原始数据, 不传映射结果
Main.qml onAstronomySunReady(json):
    backgroundManager.updateSunTimes(json.sunrise, json.sunset)
Main.qml onAstronomyMoonReady(json):
    backgroundManager.updateMoon(json.moonPhase[0].icon, json.moonPhase[0].illumination)
       ↓
===== C++ 端 =====
BackgroundManager.updateWeather(code, isDay):
    WeatherProfile profile = m_profiles.fromCode(code, isDay)   // O(1) 查表
    m_skyState.cloudCoverage = profile.cloudCoverage
    m_skyState.rainIntensity = profile.weatherParticle=="rain" ? profile.intensity : 0
    m_skyState.snowIntensity = profile.weatherParticle=="snow" ? profile.intensity : 0
    m_skyState.fogDensity    = profile.fogIntensity
    m_skyState.lightningProb = profile.lightningActive ? 1.0 : 0
    emit skyStateChanged()

BackgroundManager.updateSunTimes(sunrise, sunset):
    m_astronomyModel.setSunTimes(sunrise, sunset)
    updateAtmosphere()
    emit skyStateChanged()

updateAtmosphere():  // C++ 端计算, QML 不参与
    根据 sunProgress 计算 zenithColor/horizonColor/ambientColor/exposure/twilightFactor
    写入 m_skyState

TransitionController::onSkyStateChanged():
    比较新旧 layer 状态 → 更新 m_cloudActive/m_weatherActive/...
    触发 QProperty 信号 → QML Layer Loader.active 自动绑定更新
===== C++ 端 =====
       ↓
QML Layer ShaderEffect 只读绑定:
    CloudLayer   { active: transitionCtrl.cloudActive;   tp: transitionCtrl.cloudTP }
    WeatherLayer { active: transitionCtrl.weatherActive; tp: transitionCtrl.weatherTP }
    FogLayer     { active: transitionCtrl.fogActive;     tp: transitionCtrl.fogTP }
    LightningL   { active: transitionCtrl.lightningActive }
```

## 二、C++ 数据模型

### 2.1 WeatherProfile (天气码 → 物理状态)

```cpp
// WeatherProfile.h
#pragma once
#include <QString>
#include <QHash>

struct WeatherProfile {
    QString weatherParticle;  // "" | "rain" | "snow"
    float intensity = 0.0f;
    int weatherVariant = 0;   // rain:0普通/1雷暴/2雹/3雷暴+雹  snow:0纯/1雨夹雪
    bool cloudActive = false;
    float cloudCoverage = 0.0f;
    int cloudVariant = 0;     // 0少云/1多云/2阴
    bool fogActive = false;
    float fogIntensity = 0.0f;
    int fogVariant = 0;       // 0雾/1霾/2沙尘
    bool lightningActive = false;
};

class WeatherProfileDB {
public:
    WeatherProfileDB();  // 构造函数中初始化全部 68 码映射
    WeatherProfile fromCode(int iconCode, bool isDay) const;

private:
    // 白天映射: code → profile
    QHash<int, WeatherProfile> m_dayProfiles;
    // 夜间覆盖: 夜间晴/多云等特殊码
    QHash<int, WeatherProfile> m_nightOverrides;

    // 辅助: 强度查表 (switch-case)
    static float rainIntensity(int code);
    static float snowIntensity(int code);
    static float fogIntensity(int code);
    static int rainVariant(int code);
    static int snowVariant(int code);
    static int fogVariant(int code);
};
```

**使用**: `BackgroundManager` 持有一个 `WeatherProfileDB m_profiles` 实例，`updateWeather()` 中调用 `m_profiles.fromCode(code, isDay)` — O(1) 哈希查找。

```cpp
// BackgroundManager.cpp 关键逻辑
void BackgroundManager::updateWeather(int iconCode, bool isDay) {
    WeatherProfile p = m_profiles.fromCode(iconCode, isDay);
    m_skyState.cloudCoverage = p.cloudCoverage;
    m_skyState.rainIntensity = (p.weatherParticle == "rain")  ? p.intensity : 0;
    m_skyState.snowIntensity = (p.weatherParticle == "snow") ? p.intensity : 0;
    m_skyState.fogDensity    = p.fogIntensity;
    m_skyState.lightningProb = p.lightningActive ? 1.0f : 0.0f;
    m_skyState.starVisibility = (!isDay) ? 0.8f : 0.0f;
    emit skyStateChanged();  // ★ 触发 TransitionController + QML 绑定更新
}

void BackgroundManager::updateSunTimes(const QString &sunrise, const QString &sunset) {
    m_astronomy.setSunTimes(sunrise, sunset);
    m_astronomy.update(QDateTime::currentMSecsSinceEpoch());
    // 同步到 SkyState
    m_skyState.solarAltitude = m_astronomy.solarAltitude();
    m_skyState.solarAzimuth  = m_astronomy.solarAzimuth();
    m_skyState.moonAltitude  = m_astronomy.moonAltitude();
    m_skyState.moonAzimuth   = m_astronomy.moonAzimuth();
    m_skyState.moonPhase     = m_astronomy.moonPhase();
    m_skyState.moonIllum     = m_astronomy.moonIllum();
    updateAtmosphere();
    emit skyStateChanged();
}

void BackgroundManager::updateAtmosphere() {
    float sp = m_astronomy.sunProgress();  // 同 V2.2: 0=dawn, 0.5=noon, 1=dusk
    // 7 时段 × 3 色查表 + mix 平滑
    auto [zenith, horizon, ambient] = skyColorsForProgress(sp);
    m_skyState.zenithColor  = zenith;
    m_skyState.horizonColor = horizon;
    m_skyState.ambientColor = ambient;
    m_skyState.twilightFactor = (std::abs(sp) < 0.05f || std::abs(sp - 1.0f) < 0.05f) ? 1.0f : 0.0f;
    m_skyState.exposure = (m_astronomy.isNight()) ? 0.8f : 1.0f;
}
```

## 四、TransitionController (C++)

```cpp
// TransitionController.h
class TransitionController : public QObject {
    Q_OBJECT

    // ===== QML 绑定: 控制 Layer Loader.active =====
    Q_PROPERTY(bool cloudActive     READ cloudActive     NOTIFY cloudActiveChanged)
    Q_PROPERTY(bool weatherActive   READ weatherActive   NOTIFY weatherActiveChanged)
    Q_PROPERTY(bool fogActive       READ fogActive       NOTIFY fogActiveChanged)
    Q_PROPERTY(bool lightningActive READ lightningActive NOTIFY lightningActiveChanged)

    // ===== QML 绑定: 驱动 Layer ShaderEffect.transitionProgress =====
    Q_PROPERTY(float cloudTP     READ cloudTP     NOTIFY cloudTPChanged)
    Q_PROPERTY(float weatherTP   READ weatherTP   NOTIFY weatherTPChanged)
    Q_PROPERTY(float fogTP       READ fogTP       NOTIFY fogTPChanged)

public:
    void setSkyState(const SkyState &newState);  // 比较新旧, 编排时序

private:
    QTimer m_cloudDelay, m_weatherDelay, m_fogDelay;
    bool m_cloudActive = false, m_weatherActive = false, m_fogActive = false, m_lightningActive = false;
    float m_cloudTP = 0, m_weatherTP = 0, m_fogTP = 0;
    SkyState m_prev;
    int m_transitionId = 0;  // 防抖: 每次 transition 递增, 回调检查 ID 匹配

    void animateIn(float &tp, bool &active, QTimer &delay, int delayMs);
    void animateOut(float &tp, bool &active, int delayMs);
    void abortAll();  // 中断当前所有过渡
};
```

```cpp
// TransitionController.cpp 时序逻辑
void TransitionController::setSkyState(const SkyState &s) {
    ++m_transitionId;  // 防抖标记
    SkyState prev = m_prev;
    m_prev = s;

    // === Cloud ===
    bool cloudNow = s.cloudCoverage > 0.01f;
    if (cloudNow && !prev.cloudCoverage) {
        // 激活: delay 100ms → animate cloudTP 0→1 over 600ms
        animateIn(m_cloudTP, m_cloudActive, m_cloudDelay, 100);
    } else if (!cloudNow && prev.cloudCoverage > 0.01f) {
        animateOut(m_cloudTP, m_cloudActive, 600);
    } else if (cloudNow && std::abs(s.cloudCoverage - prev.cloudCoverage) > 0.01f) {
        // 同 Layer 覆盖率变化 → QML 端 Behavior on cloudCoverage 平滑 (无需 C++ 干预)
    }

    // === Weather ===
    bool weatherNow = s.rainIntensity + s.snowIntensity > 0.001f;
    if (weatherNow && !(prev.rainIntensity + prev.snowIntensity > 0.001f)) {
        int delay = cloudNow ? 400 : 100;  // 有云时等云先入场
        animateIn(m_weatherTP, m_weatherActive, m_weatherDelay, delay);
    } else if (!weatherNow && (prev.rainIntensity + prev.snowIntensity > 0.001f)) {
        animateOut(m_weatherTP, m_weatherActive, 600);
    }

    // === Fog ===
    bool fogNow = s.fogDensity > 0.001f;
    if (fogNow && !(prev.fogDensity > 0.001f)) {
        animateIn(m_fogTP, m_fogActive, m_fogDelay, 100);
    } else if (!fogNow && prev.fogDensity > 0.001f) {
        animateOut(m_fogTP, m_fogActive, 400);
    }

    // === Lightning ===
    m_lightningActive = s.lightningProb > 0.0f;
    emit lightningActiveChanged();
}

void TransitionController::animateIn(float &tp, bool &active, QTimer &delay, int delayMs) {
    int localId = m_transitionId;
    QTimer::singleShot(delayMs, this, [=, &tp, &active]() {
        if (m_transitionId != localId) return;  // 已被中断, 放弃
        active = true;
        emit /* activeChanged */;
        // tp 由 QML Behavior 驱动: 设置 target=1.0 → NumberAnimation 自动平滑
        tp = 1.0f;
        emit /* tpChanged */;
    });
}

void TransitionController::abortAll() {
    ++m_transitionId;  // 使所有悬而未决的 callback 失效
    // 恢复当前 active 层 tp=1.0
    if (m_cloudActive)   m_cloudTP = 1.0f;
    if (m_weatherActive) m_weatherTP = 1.0f;
    if (m_fogActive)     m_fogTP = 1.0f;
}
```

**Behavior 驱动 tp 平滑**（在 Layer QML 内部）：

```qml
// CloudLayer.qml
ShaderEffect {
    property real transitionProgress: 0.0
    Behavior on transitionProgress { NumberAnimation { duration: 600; easing.type: Easing.OutCubic } }
    // ...
}
```

C++ 端设 `tp = 1.0f`，QML 的 `Behavior on transitionProgress` 自动驱动 0→1 平滑动画。

**时序编排总结**：

| 过渡场景 | SkyLayer | CloudLayer | WeatherLayer | FogLayer |
|------|:---:|------|------|------|
| Sunny → Cloudy | 天空渐灰(300ms平滑) | +100ms fadeIn 600ms | — | — |
| Sunny → Rain | 天空暗化 | +100ms fadeIn | +300ms fadeIn 600ms | — |
| Rain → Snow | — | — | fadeOut→换源→fadeIn | — |
| Day → Night | sunProgress 驱动天空渐变 | 保持 | 保持 | 保持 |

## 五、各 Layer 设计

### 5.1 SkyLayer (`shaders/sky.frag`)

```
职责:    天空渐变 + 太阳光晕 + 月亮SDF + 星星
始终渲染: ✅
粒子循环: 50 (星星 only, starVisibility>0 时)
```

| uniform | 类型 | 来源 |
|------|:---:|------|
| time | float | GlobalClock.elapsed |
| solarAltitude, solarAzimuth | float×2 | AstronomyModel |
| moonAltitude, moonAzimuth, moonPhase, moonIllum | float×4 | AstronomyModel |
| zenithColor, horizonColor, ambientColor | vec4×3 | Manager.updateAtmosphere() |
| exposure, twilightFactor | float×2 | Manager.updateAtmosphere() |
| starVisibility | float | SkyState |
| parallaxX, parallaxY | float×2 | WeatherBackground.MouseArea |

**不需要 `transitionProgress`** — SkyLayer 始终渲染，变化通过 `sunProgress` (60s Timer) 和属性级 `Behavior` 平滑。

**太阳光晕**：根据 `solarAltitude` 计算屏幕位置和颜色（低角→橙红，高角→亮白）。

**月亮**：根据 `moonAltitude/Azimuth` 计算位置，`moonPhase` SDF 裁剪月牙。

**星星**：50 颗随机位置 + `sin(time*2+seed)` 闪烁，`starVisibility` 控制亮度。

### 5.2 AtmosphereLayer (`shaders/atmosphere.frag`)

```
职责:    twilight 散射色调 + 地平线渐变 + 曝光控制
始终渲染: ✅
粒子循环: 0
```

| uniform | 类型 | 来源 |
|------|:---:|------|
| time | float | GlobalClock |
| zenithColor, horizonColor, ambientColor | vec4×3 | Manager |
| exposure, twilightFactor | float×2 | Manager |

不引入额外纹理，仅对 SkyLayer 的输出做色调调整。可以省略为一个简单的后处理 pass。

### 5.3 CloudLayer (`shaders/cloud.frag`)

```
职责:    2~3层 fBM 云噪声漂移
条件渲染: cloudCoverage > 0.01
粒子循环: 0
```

| uniform | 类型 | 来源 |
|------|:---:|------|
| time | float | GlobalClock |
| cloudCoverage | float | SkyState |
| cloudOpacity | float | ≈1.0 |
| variant | int | Profile (0少云/1多云/2阴) |
| windSpeed | float | Manager |
| transitionProgress | float | TransitionController.cloudTP |

```
transitionProgress (tp):
  tp=0 → 云在屏幕右侧外(cloudX+0.6), 覆盖率=0
  tp=1 → 云在正常位置, 覆盖率=cloudCoverage
```

### 5.4 RainLayer (`shaders/rain.frag`)

```
职责:    雨丝粒子 + 闪电指示
条件渲染: rainIntensity > 0.001
粒子循环: MAX 60 (分档)
```

| uniform | 类型 | 来源 |
|------|:---:|------|

## 七、ShaderEffect 统一模板

```qml
// 以 CloudLayer.qml 为例
ShaderEffect {
    id: effect
    anchors.fill: parent
    opacity: 1.0

    // === 通用 ===
    property real time: GlobalClock.elapsed

    // === Layer 专属 ===
    property real cloudCoverage: 0.0
    property real cloudOpacity: 1.0
    property int variant: 1
    property real windSpeed: 0.0
    property real transitionProgress: 0.0

    fragmentShader: "qrc:/shaders/cloud.frag.qsb"

    // === 容错 ===
    Rectangle {
        anchors.fill: parent
        visible: effect.status !== ShaderEffect.Compiled
        color: "#8899aa"
        Component.onCompleted: {
            if (effect.status === ShaderEffect.Error)
                console.warn("CloudLayer: ShaderEffect Error - using fallback")
        }
    }
}
```

**关键点**：
- `time` 绑定 `GlobalClock.elapsed`，不维护自己的 `NumberAnimation`
- `fragmentShader` 指向 `.qsb` 文件（由 `qt6_add_shaders` 自动生成）
- 容错 `status !== Compiled` + `console.warn` 日志
- 所有 Layer 的 QML 模板结构一致，只替换属性和 shader 路径

## 八、Shader Uniform 映射规范（Qt6 标准）

- `qt_Matrix` 和 `qt_Opacity` **必须在 uniform block 最前面**（offset 0 和 64），名称固定
- 之后的属性按**名称**匹配（非声明顺序）—— QML `property real foo` 匹配 GLSL `float foo`
- 所有 uniform 必须在 `layout(std140, binding=0)` block 内，不允许 block 外变量
- `color` → `vec4`（已 premultiplied），`real` → `float`，`int` → `int`（Qt6 直接支持，不需要 float→int hack）
- 纹理用 `layout(binding=1) uniform sampler2D tex;`（binding=0 保留给 uniform block）

## 十、性能策略

### 10.1 分档粒子上限

```qml

```

| 档位 | Rain 粒子 | Snow 粒子 | Sky 星星 | 云 noise octaves |
|:---:|:---:|:---:|:---:|:---:|
| 高(2) | 60 | 60 | 50 | 5 |
| 中(1) | 40 | 40 | 35 | 3 |
| 低(0) | 20 | 20 | 20 | 2 |

### 10.2 优化要点

- 循环上限编译时常量 (`MAX_DROPS=60`)，uniform break 控制实际数量 — 零 warp divergence
- 过渡期间 tp=0 时粒子循环整帧跳过 (`activeIntensity < 0.001 → return`)
- 天文计算：`AstronomyModel.update()` 每 60s 一次，日出日落前后 5min 缩短为 10s
- 常驻 Layer：SkyLayer + AtmosphereLayer (2 shader)，其余按需激活
- 过渡期峰值：最多全部 6 Layer 激活，但每个轻量（单粒子循环上限 60 × 共用）

### 10.3 性能基准目标

| 配置 | 分辨率 | 目标 |
|------|------|:---:|
| 桌面独显 (GTX 1060+) | 1920×1080 | ≥ 60 fps |
| 桌面集显 (Intel UHD) | 1920×1080 | ≥ 30 fps |
| 笔记本 (Intel Iris) | 1920×1080 | ≥ 30 fps |
| 4K 显示器 | 3840×2160 | ≥ 30 fps (PerfLevel=中或低) |
