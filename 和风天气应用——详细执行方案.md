### 阶段 0 成果总结

**C++ 后端（weatherApi.h / .cpp）**

```
架构：
  QML 调 Q_INVOKABLE
    → 请求函数查缓存（命中则直接调 handler 返回）
    → 未命中则 sendRequest(url, type) 发 HTTP
    → onReplyFinished 收回复 → 写缓存 → switch(type) 分发 handler
    → handler 解析 JSON → emit 信号 → QML 接收

信号设计：
  对象型信号 → 注入 _location 字段标识来源城市
  数组型信号 → 首个参数为 loc（城市 ID）
  
缓存策略（WeatherCache）：
  数据库：~/.qweather_cache.db（SQLite）
  表结构：k TEXT PRIMARY KEY, data TEXT, ts INTEGER
  TTL 控制每条数据的有效期
```

### 2.4 数据流设计

```
DashboardPage：
  Component.onCompleted → weatherApi.topCity("cn", 4)
  cityTopReady → cardModel 填充 → 逐个 weatherApi.weatherNow(id)
  weatherNowReady → 更新对应卡片 temp/icon

AirQualityPage：
  Component.onCompleted → 复用 cardModel 的 id 列表 → 逐个 weatherApi.airCurrent(lat, lng)
  airCurrentReady → 更新对应卡片 AQI

ForecastPage：
  同上，调 weatherApi.weatherDaily("3d", id)
  提供用户控件切换 3d/7d/10d

AstroSolarPage：
  同上，调 weatherApi.astronomySun + astronomyMoon + solarRadiation

CityDetailPage：
  接收 cityId 参数 → 一次性调多个 API
  weatherNow + weatherDaily + weatherHourly + airCurrent + indices + astronomySun
  TabBar 切换显示不同数据类型

FavoritesPage：
  本地 ListModel 管理，右键删除
  双击跳转 CityDetailPage

SettingsPage：
  缓存管理（调用 WeatherCache.cleanExpired / clearAll）
  主题切换占位
  温度单位占位
```

### 2.6 CityCard 改造

将 CityCard 改造为通用组件，通过 `cardType` 属性控制显示内容：

```
cardType: "temperature" | "aqi" | "forecast" | "astro"
↓
根据类型显示不同的数据字段：
  temperature → 图标 + 温度
  aqi        → AQI 数值 + 等级色环 + 首要污染物
  forecast   → 日期 + 白天/夜间图标 + tempMax/tempMin
  astro      → 日出/日落时间 + 月相图标
```
