### 阶段 0 成果总结

**C++ 后端（weatherApi.h / .cpp）**

```
架构：
  QML 调 Q_INVOKABLE
    → 请求函数查缓存（命中则直接调 handler 返回）
    → 未命中则 sendRequest(url, type) 发 HTTP
    → onReplyFinished 收回复 → 写缓存 → switch(type) 分发 handler
    → handler 解析 JSON → emit 信号 → QML 接收

信号设计：
  对象型信号 → 注入 _location 字段标识来源城市
  数组型信号 → 首个参数为 loc（城市 ID）
  
缓存策略（WeatherCache）：
  数据库：~/.qweather_cache.db（SQLite）
  表结构：k TEXT PRIMARY KEY, data TEXT, ts INTEGER
  TTL 控制每条数据的有效期
```

### 2.5 各页面数据流

**DashboardPage（城市来源：历史记录 → 热门城市 fallback）**

```
Component.onCompleted
  → WeatherCache.get("dashboard_cities")   // 读取历史城市列表
  → 如果历史城市 < 4：补调 weatherApi.topCity("cn", 4)
  → 逐个 weatherApi.weatherNow(id)
  → weatherNowReady → 更新对应卡片

用户搜索城市
  → SearchBar → cityLookupReady → 点选城市
  → 该城市加入历史记录 → WeatherCache.set("dashboard_cities", cityIds)
  → 刷新卡片 → weatherApi.weatherNow(newCityId)
```

> **没有 AddCityButton**。城市卡片由搜索行为自动维护历史列表填充。

**AirQualityPage / ForecastPage / AstroSolarPage**

```
Component.onCompleted
  → 复用 DashboardPage 的城市列表（从全局 store 读取）
  → 逐个调对应 API
  → 信号回来更新卡片

ForecastPage 额外：
  → 顶部选择器切换 3d/7d/10d/15d/30d
  → 切换时重新调 weatherApi.weatherDaily("{n}d", id)
```

**CityDetailPage（接收 cityId，默认 = root.focusCityId）**

```
进入页面 → 一次性调 6 个 API：
  weatherNow + weatherDaily("7d") + weatherHourly("24h")
  + airCurrent + indices("3d","0") + astronomySun + astronomyMoon

TabBar: [ 实时 | 逐日 | 逐时 | 空气 | 指数 | 天文 ]
  → SwipeView 联动切换
```

**FavoritesPage**

```
ListModel 存储：{ name, id, pinned }
  双击 → 跳转 CityDetailPage(cityId)
  右键 → 删除
  数据持久化到 QSettings（跨会话保留）
```

**SettingsPage**

```
├── 主题：Light / Dark（Switch，阶段 4 实现）
├── 缓存：清除缓存 + 显示大小
├── 默认城市数：Slider 2-6
├── 温度单位：°C / °F（预留）
└── 关于：v0.1 + 数据来源和风天气
```

### 2.7 全局数据：CityStore 单例

为避免每个页面重复维护 cityList，引入 `CityStore` 集中管理：

```qml
// CityStore.qml (pragma Singleton)
QtObject {
    property var trackedCities: []       // [{ name, id, lat, lon }]，最多 6 个
    property string focusCityId: ""
    property string pinnedCityId: ""     // 城市分区固定的城市

    function addCity(cityObj) { ... }    // 去重追加
    function removeCity(id) { ... }
    function pinCity(id) { ... }
}
```

各页面从 CityStore 读取城市列表，调用对应 API，不再各自维护 cardModel。
