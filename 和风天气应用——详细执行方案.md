## 实际架构（与原始计划的关键差异）

> 原始阶段 2 计划（2026-07-11）设计了 QML 端 `CityStore` 单例 + 各页面自行管理数据。实际实施中（7/13）全面转向 **C++ Store + QML 纯渲染** 模式，架构更健壮。

### 架构对比

| 方面 | 原始计划 | 实际实现 |
|------|---------|---------|
| 数据层 | QML `CityStore.qml` pragma Singleton | C++ `ForecastStore` / `AirQualityStore` / `SolarAstronomyStore` / `CityDetailStore` |
| 城市列表 | CityStore 集中管理 | `Main.qml` `cityList` → `onCityListChanged` → 同步到各 C++ Store |
| 卡片组件 | 1 个 `CityCard` 通过 `cardType` 切换 4 种模式 | 4 个独立专用卡片：`CityCard` / `AirQualityCard` / `ForecastCard` / `SolarAstronomyCard` |
| 焦点城市 | `CityStore.focusCityId` | `Main.qml` `focusId` → 同步到 `cityDetailStore.setCity()` |
| 仪表盘 | 独立 `DashboardPage.qml` | 仍内联在 `Main.qml` 中（未拆分） |
| 图表 | 无 | `ForecastCard` 内嵌 `ChartView`（QtCharts），预声明 SplineSeries + LineSeries |

### C++ Store 数据流模式（统一范式）

```
Main.qml cityList 变更
  → onCityListChanged
    → store.cities = root.cityList (各 Store 同步)
      → Store::setCities() → refreshAll()
        → 逐个调用 weatherApi.xxx()
          → API 信号返回 → Store handler 解析 JSON
            → 写入 QVariantMap（每次创建新对象触发 QML 绑定）
              → emit dataChanged()
                → QML Card 绑定自动刷新
```

## UI 设计规范

> 确立于 2026-07-13，基于实际代码 + todo.md 界面设计要求提炼
> 绿色渐变背景为临时方案，正式背景为动态天气背景（ShaderEffect）

### 核心架构

```
Layer 0: WeatherBackground (ShaderEffect)  ← 晴天/阴天/雨天/雪天/夜间动态切换 + 鼠标视差
Layer 1: 半透明蒙层（增强文字可读性）
Layer 2: UI 前景（SidePanel + StackLayout 卡片）
```

背景实现路径：Rectangle 渐变（当前） → Canvas 粒子（备选） → ShaderEffect GPU（最终）

### 风格：Fluent Glassmorphism

半透明 ARGB 叠层 + `border.width: 1` 弱边框。无 backdrop-blur、无 DropShadow。

### 颜色

| 角色 | 值 | 说明 |
|------|-----|------|
| 卡片底色 | `#20ffffff` | 透出动态背景，全天气类型适用 |
| 卡片边框 | `#30ffffff` | hover `#50ffffff` |
| 深色遮罩 | `#20000000` / `#1a000000` | InfoChip / 侧边栏 |
| 文字主 | `white` | 最高对比度 |

### 卡片 Hover 标准

> 确立于 2026-07-14，统一所有卡片 hover 行为

**数值：**

```
背景:  #20ffffff → #25ffffff  (+5%)
边框:  #30ffffff → #50ffffff  (+20%)
过渡:  200ms ColorAnimation (color + border.color)
```

**QML 模板：**

```qml
color: <mouseId>.containsMouse ? "#25ffffff" : "#20ffffff"
border.width: 1
border.color: <mouseId>.containsMouse ? "#50ffffff" : "#30ffffff"

Behavior on color        { ColorAnimation { duration: 200 } }
Behavior on border.color { ColorAnimation { duration: 200 } }
```

**三类卡片 MouseArea 放置策略：**

| 类型 | MouseArea 方式 | 额外处理 |
|------|---------------|---------|
| 有点击交互 (CityCard/FavoritesPage) | Rectangle 内置 MouseArea 同时处理 click + hover | 无 |
| 纯展示 (AirQuality/SolarAstronomy) | 外层 Item 放独立 MouseArea `{ hoverEnabled: true }` | 无 |
| 含交互子元素 (ForecastCard) | 外层 Item 放独立 MouseArea | `acceptedButtons: Qt.NoButton` |

**不变项（不参与 hover）：**

```
强调色:     #4caf50 (焦点高亮，不参与 hover)
```

## 全局数据流（实际）

### 焦点城市行为（Main.qml）

```
用户首次打开 → 无历史 → weatherApi.topCity("cn", 4) → cityList 填充热门城市
  → focusId = cityList[0].id → onFocusIdChanged → addHistoryEntry + weatherNow + cityDetailStore.setCity

单击 CityCard → root.addFocus(cityId) → 已在列表则只切焦点 / 不在则替换焦点城市卡片
双击 CityCard → focusId = cityId + stack.currentIndex = 5 (CityDetailPage)
SearchBar 选择 → root.addFocus(cityId, name, lat, lon) → 新城市替换当前焦点城市卡片

onFocusIdChanged:
  → addHistoryEntry(focusId) → 写入历史(去重+LIFO+上限20) → saveHistory()
  → weatherApi.weatherNow(focusId) → 刷新实时天气
  → cityDetailStore.setCity(...) → 城市详情页同步
```

### 侧边栏城市分区 ID 优先级

```
pinned[0].id > favorites[0].id > history[0].id > focusId
```

### 收藏持久化

```
收藏/固定/历史 → JSON.stringify → weatherCache.save("fav_cities"/"pin_cities"/"hist_cities", json)
启动时 → weatherCache.load(...) → JSON.parse → 恢复到对应数组
```

## 踩坑记录汇总

### Qt/C++ 层

| 问题 | 现象 | 根因 | 解决方案 |
|------|------|------|---------|
| QGuiApplication 崩溃 | SIGSEGV | QtCharts 内部用 QGraphicsTextItem→QWidgetTextControl，依赖 QtWidgets | QGuiApplication → QApplication |
| DateTimeAxis 不渲染 | 折线不显示 | Qt 6 Charts QML DateTimeAxis + 预声明 series 不兼容 | 改用 ValueAxis + 序号索引 |
| `clear()` 后轴不自动缩放 | 折线压在底部 | Qt Charts: clear() 后 axis 范围保持不变 | 先算范围→setRange→clear→append |
| `removeAllSeries()` + `createSeries()` 产生 NaN | Ignored NaN 警告泛滥 | 清理/重建 series 导致轴进入无效状态 | 预声明 series + clear() + append() |
| C++ `QDateTime` 时区 | X 轴显示大数字 | `fromString("yyyy-MM-dd")` 默认 LocalTime | 转为序号索引后不再使用 |
| QML `property var` 绑定不更新 | 卡片无数据 | JS 对象原地修改引用不变，QML Changed 信号不触发 | C++ 端每次创建新 QVariantMap |
| `_loc` vs `location` 路由冲突 | 天文学 API 无响应 | `onReplyFinished` 中 `q.queryItemValue("location")` 对天文 API 返回 "lat,lng"，导致 cityId 路由失败 | `_loc` 优先提取 |
| astronomySun/Moon 请求不返回 | solarReady 正常但 sunReady/moonReady 日志不出现 | Store 传 "lat,lng" 格式给 `/v7/astronomy/*`，devapi 代理不支持 | 改用 LocationID（cityId） |

### QML 层

| 问题 | 现象 | 根因 | 解决方案 |
|------|------|------|---------|
| Grid polish loop | 控制台刷屏 | Grid 子项 `width: parent.width / N` 形成循环依赖 | 子项宽度引用卡片级 ID |
| QML Layout 内使用 anchors | 警告刷屏 | ColumnLayout/RowLayout 子项使用 `anchors` | 替换为 `Layout.alignment` |
| `visible` 绑定返回 undefined | 卡片错误显示 + TypeError | `detail.now && detail.now.temp` 中 undefined 时返回 undefined 非 false | 所有 guard 使用 `!!` 强制布尔化 |
| `visible: false` 卡片内绑定仍求值 | TypeError 刷屏 | QML 始终评估所有已实例化组件的绑定 | C++ 端预初始化所有嵌套 map/list 为空对象 |
| `labelFormat: "%.0f°"` 乱码 | Y 轴显示 `?C` | Windows 字体不支持 `°` 字符 | 去掉特殊字符 |
| `pollutants && length > 0` 返回 undefined | 无法赋值 bool | undefined && x 返回 undefined | 改为 `!!pollutants && ...` |

### 设计教训

1. **C++ 数据预处理 > QML JS**：JSON 解析、日期转换、坐标计算移到 C++，消除 ReferenceError/类型错误/信号丢失一整类 bug
2. **Qt Charts 必须 QApplication**：内部依赖 QWidgetTextControl(Graphics View)
3. **预声明 series 优于 createSeries**：ChartView + 预声明 children 比动态创建稳定
4. **QML 属性绑定 ≠ JS 深比较**：`property var` 依赖引用判等，深层修改不触发 Changed 信号 → C++ 端每次创建新对象
5. **先设轴范围后清数据**：clear() 后 axis 状态失效，提前设 range 避免 NaN 间隙
6. **`_loc` 优先于 `location` 提取路由键**：当 API 同时具有 location（业务参数）和 `_loc`（追踪参数）时，追踪参数优先
7. **新增端点优先复用已验证的参数格式**：ForecastStore 用 LocationID 始终正常，新端点不应假设文档中所有格式在代理环境下都能工作
8. **多路异步 API 用中间累积 → 统一 emit 模式**：`m_solarRaw` 中间缓冲 + `mergeAndEmit` 合并写入
9. **`undefined && x` 返回 `undefined` 非 `false`**：QML 中所有 visible guard 表达式必须用 `!!` 强制返回布尔值
10. **C++ 端预初始化空 map/list**：在构造函数和 setCity() 两处执行，因为 QML 绑定在首次数据到达前就已开始求值
