# 天气模拟系统 —— 需求文档

> 基于 `indicationsForDynamic_background.md` 通读提炼，最终确定要实现的完整需求。
> 版本：V2.2

---

## 一、目标

在现有天气应用中实现一个**天气+天文天空模拟系统**，替代临时绿色渐变背景。

- 根据焦点城市的天气码动态切换背景效果
- 根据日出日落时间 + 当前时间驱动太阳/月亮位置与天空色调
- 过渡不是两个画面切换，而是元素级别过程式动画（云飘入、雨渐密、光晕淡出）
- 提供调试面板，可手动接管所有参数进行测试

---

## 二、架构

```
Main.qml
├── WeatherBackgroundManager    ← 中间层（单一状态源）
│     模式: Auto(跟随App天气) | Debug(调试面板接管)
│     输出: 天气参数(type/intensity/variant/windSpeed) + 天文参数(25 uniform)
│
├── WeatherBackground           ← 渲染容器
│     ├── Loader A             ← 双层 + bool角色交换(零开销reload)
│     ├── Loader B             ← 每层独立 transitionProgress 动画 (800ms)
│     ├── NumberAnimation×2    ← 驱动进入层 0→1 + 退出层 1→0
│     ├── abortCurrentTransition() ← 中断式防抖
│     └── MouseArea            ← 鼠标视差 parallaxX/Y
│
├── SunnyBg | CloudyBg | RainBg | SnowBg | NightBg | FogBg  ← 6个 ShaderEffect
│
├── [现有 UI: RowLayout + StackLayout]
│
└── WeatherBackgroundDebugPanel ← 浮动面板 (Ctrl+Shift+B)
      Auto/Debug 模式切换 + 天气类型网格 + 强度/变体/风速滑块
```

**规则**：
- App 只通过 `Manager.updateAppWeather()` 改背景
- DebugPanel 只通过 `Manager.setDebugXxx()` 接管背景
- 两者互不感知，Manager 是唯一真值源

## 三、过渡机制：三层过渡模型

### 问题

68 种天气码产生 2278 种过渡对。只用跨类型 800ms 过程式过渡只能覆盖 ~30 种类型对，**同类型内的 intensity/variant 变化（如 305小雨→307大雨）会瞬间跳变**，视觉不连贯。

### 三层过渡

```
switchTo(requestedType):
    sameType ← (requestedType == activeType)
    paramsChanged ← (intensity≠ || variant≠)

    ┌─ sameType && !paramsChanged  → return (NO-OP)         // 层0: 相同不变
    ├─ sameType && paramsChanged   → smoothParamsTransition() // 层1: 参数平滑
    ├─ !sameType && transitioning  → abort + restart
    └─ !sameType                   → startTypeTransition()   // 层2: 过程式过渡
```

### 层 1：参数平滑过渡（新增）

同类型参数变化时，不换 Loader，在当前活跃 shader 上用 NumberAnimation 平滑：

```
smoothParamsTransition(newIntensity, newVariant):
    anim.from = shader.intensity,  anim.to = newIntensity
    anim.duration = 400ms,  easing = InOutCubic
    anim.start()
    // variant 同理平滑
```

视觉效果：雨滴密度缓缓增大/减小，闪电从无到偶尔再到频繁。

### 层 2：过程式过渡（现有）

跨类型变化时，双层 Loader + transitionProgress 驱动：

| 元素 | 进入 (0→1) | 退出 (1→0) |
|------|-----------|-----------|
| 云层 | 从屏幕右侧外漂移进来 | 漂出左边界 |
| 雨滴 | 从 0 条逐渐增多 | 逐渐减少至 0 |
| 光晕 | 从不可见渐亮 | 淡出消失 |
| 天空色 | 从对方天色渐变到本方天色 | 反向 |

800ms InOutCubic，两层同时驱动。过渡完成 → 角色交换 → 卸载旧背景。

### 覆盖矩阵

| 过渡类型 | 举例 | 机制 | 视觉 | 时长 |
|------|------|------|------|:---:|
| 完全相同 | 305→305 | NO-OP | 无 | — |
| 同类型±intensity | 305→307 | 参数平滑 | 雨滴密度缓缓增大 | 400ms |
| 同类型±variant | 307→302 | 参数平滑 | 闪电从无到偶尔 | 400ms |
| 同类型±两者 | 305→303 | 参数平滑 | 密度↑+闪电渐現 | 400ms |
| 跨类型 | 305→400 | transitionProgress 过程式 | 雨丝渐消+雪花渐显 | 800ms |

**2278 种过渡对全部覆盖，全部分配到合适的过渡机制。**

### 防抖

过渡期间再次切换 → `abortCurrentTransition()` → 恢复当前活跃层完整状态 → 再发起新过渡。同类参数动画中断同理：stop anim → set 当前值 → 开始新 anim。

## 四、天气码映射

### 4.1 68 个天气码 → 6 种背景

| 代码范围 | 数量 | 背景类型 |
|---------|:---:|:---:|
| 100 | 1 | sunny |
| 101~104 | 4 | cloudy |
| 150~153 | 4 | night（isDay=false 全部码也归 night） |
| 300~399 | 22 | rain |
| 400~499 | 15 | snow |
| 500~515 | 14 | fog |
| 900~999 | 3 | sunny（默认回退） |

### 4.2 intensity（粒子密度/覆盖率，0.0~1.0）

| 类型 | 低 (0.1-0.3) | 中 (0.4-0.6) | 高 (0.7-1.0) |
|------|------|------|------|
| rain | 毛毛雨/小雨 | 中雨/大雨 | 暴雨/大暴雨/特大暴雨 |
| snow | 小雪 | 中雪/大雪 | 暴雪 |
| fog | 薄雾 | 雾/霾/沙尘 | 浓雾/强沙尘暴 |
| cloudy | 少云 | 多云/晴间多云 | 阴 |

### 4.3 variant（变体，0~3）

| 类型 | 0 | 1 | 2 | 3 |
|------|:---:|:---:|:---:|:---:|
| rain | 普通雨 | 雷暴（闪电闪烁） | 冰雹/冻雨（硬粒） | 雷暴+冰雹 |
| snow | 纯雪 | 雨夹雪 | — | — |
| fog | 雾（乳白） | 霾（黄色调） | 沙尘（棕褐调） | — |
| cloudy | 少云 | 多云 | 阴 | — |

**68 码 → (type, intensity, variant) 三元组，零信息损失。**

## 五、天文天空模型

### 5.1 核心参数

```
sunProgress = (当前分钟 - 日出分钟) / (日落分钟 - 日出分钟)
// 0=日出, 0.5=正午, 1=日落, <0=日出前, >1=日落后
```

### 5.2 天空色模型（7时段×3色）

| 时段 | sunProgress | 天顶 | 地平线 | 下方 |
|------|:---:|------|------|------|
| 深夜 | < -0.3 | `#0a0a1a` | `#0d0d28` | `#050510` |
| 蓝调(日出前) | -0.3~0 | `#1a2a5a` | `#4a3060` | `#0a0a20` |
| 金粉(日出) | 0~0.15 | `#4a6fa5` | `#f4a460` | `#e07050` |
| 白天 | 0.15~0.7 | `#4a90d9` | `#87ceeb` | `#c8e0f0` |
| 暖午后 | 0.7~0.9 | `#5a8ab5` | `#d4996a` | `#c0b0a0` |
| 橙红(日落) | 0.9~1.0 | `#3a5a8a` | `#e07840` | `#a03030` |
| 紫调(日落后) | 1.0~1.3 | `#2a1a4a` | `#602040` | `#0a0a20` |

各时段间 `mix()` 平滑过渡。天气类型在此基础上调整饱和度亮度。

### 5.3 太阳

```
angle = (sunProgress - 0.5) × π
sunX = 0.5 + sin(angle) × 0.55     // 左→中→右
sunY = 0.7 - cos(angle) × 0.5      // 地平→天顶→地平
elevation = cos(angle)              // 0=地平线, 1=天顶
sunColor = mix(#ff6633, #fffff0, √elevation)
sunGlow = 0.04 + (1-elevation)×0.15
```

### 5.4 月亮

- 位置：与太阳对跖（相位差半圈）
- 可见性：仅夜晚+twilight淡入淡出
- 月相：800(新月)~807(残月)，用 SDF 裁剪月牙形状
- 数据源：`weatherApi.astronomySun/Moon`

### 5.5 数据流

```
focusId变更 → astronomySun/Moon API → Manager.receiveSun/Moon
Timer(60s) → Manager.updateSunSkyParams() → 计算25个uniform
→ Qt.binding → ShaderEffect uniforms 自动更新
```

## 六、6 种 ShaderEffect 背景

| 背景 | shader | 核心效果 | 粒子循环 | 天文融合 |
|------|--------|---------|:---:|------|
| SunnyBg | sunny.frag | 时变三色渐变 + 太阳光晕 + 光线 + 噪声纹理 | 0 | 太阳位置/颜色 |
| CloudyBg | cloudy.frag | 时变三色 + 2~3层云噪声漂移 + variant覆盖率 | 0 | 漫射光(云遮挡) |
| RainBg | rain.frag | 时变三色×暗化 + 雨丝粒子 + 闪电 + 冰雹 + 涟漪 | 60 | 太阳不可见 |
| SnowBg | snow.frag | 时变三色×冷化 + 雪花飘落 + 风吹摆动 + 雨夹雪 | 60 | 漫射光晕 |
| NightBg | night.frag | 深蓝黑 + 星空闪烁 + 月牙SDF | 50 | 月亮渲染 |
| FogBg | fog.frag | 时变三色×去饱和 + 雾气带 + 霾/沙尘色调 | 0 | 太阳不可见 |

每个 ShaderEffect 含 fallback Rectangle 渐变（ShaderEffect.Error 时显示）。

## 七、Shader Uniform 接口（25个）

```glsl
layout(std140, binding=0) uniform buf {
    mat4 qt_Matrix;       float qt_Opacity;
    // 动画 (2):  time, transitionProgress
    // 天气 (3):  intensity, windSpeed, variant
    // 天文 (9):  sunProgress, sunX, sunY, sunColor, sunGlow,
    //            skyTop, skyHorizon, skyBottom, isNight, isTwilight
    // 月亮 (5):  moonVisible, moonX, moonY, moonPhase, moonIllum
    // 交互 (2):  parallaxX, parallaxY
};
```

## 九、性能策略

| 档位 | Rain粒子 | Snow粒子 | Night星星 | 适用 |
|:---:|:---:|:---:|:---:|------|
| 高(2) | 60 | 60 | 50 | 桌面独显 |
| 中(1) | 40 | 40 | 35 | 集显/笔记本 |
| 低(0) | 20 | 20 | 20 | 嵌入式 |

- 循环上限编译时常量(60)，uniform break 控制实际数量，零 warp 发散
- 过渡期间 tp=0 层自动跳过粒子循环，不翻倍 GPU 负载
- 天文计算每 60s 一次 <0.1ms
- `AppSettings.perfLevel` 运行时调节
